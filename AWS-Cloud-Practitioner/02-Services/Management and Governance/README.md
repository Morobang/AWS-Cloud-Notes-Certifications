# Management and Governance — Category Overview

Management and Governance services help teams monitor infrastructure, enforce compliance, automate operations, and manage AWS accounts at scale. This is one of the most heavily tested categories on the Cloud Practitioner exam.

---

## Services in this category

| Service | Purpose | Key concept |
|---------|---------|-------------|
| AWS Auto Scaling | Automatically adjust resource capacity | Scale in/out based on demand |
| Amazon CloudWatch | Metrics, logs, and alarms | Monitor infrastructure and applications |
| AWS CloudFormation | Infrastructure as Code (IaC) | Deploy resources from templates |
| AWS CloudTrail | API call logging | Who did what, when, from where |
| AWS Compute Optimizer | Resource right-sizing recommendations | Reduce over-provisioned resources |
| AWS Config | Resource configuration history and compliance | Track config changes, enforce rules |
| AWS Control Tower | Multi-account governance setup | Automated landing zone setup |
| AWS Health Dashboard | AWS service health events | Know when AWS services are degraded |
| AWS License Manager | Software license tracking | Manage BYOL licenses across AWS |
| AWS Management Console | Web-based GUI for AWS | Browser interface for all services |
| AWS Organizations | Multi-account management | Centralized billing, SCPs, account structure |
| AWS Service Catalog | Pre-approved product catalog | Self-service provisioning within guardrails |
| AWS Systems Manager | Operational management of EC2 and on-prem | Patch, run commands, manage parameters |
| AWS Trusted Advisor | Best practice checks | Cost, security, performance, fault tolerance |
| AWS Well-Architected Tool | Architecture review against AWS pillars | Review workloads against best practices |
| Service Quotas | View and request quota increases | Manage service limits |
