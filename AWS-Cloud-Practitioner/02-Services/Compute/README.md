# Compute Services

AWS compute services run your code — from long-running virtual machines to serverless functions to batch jobs.

## Services in this category

| Service | What it does | Key use case |
|---------|-------------|--------------|
| [Amazon EC2](./Amazon%20EC2.md) | Virtual servers in the cloud | Full control over OS and environment |
| [AWS Elastic Beanstalk](./AWS%20Elastic%20Beanstalk.md) | PaaS for web applications | Deploy apps without managing infrastructure |
| [Amazon Lightsail](./Amazon%20Lightsail.md) | Simple VPS with fixed pricing | Simple websites, small applications |
| [AWS Batch](./AWS%20Batch.md) | Managed batch computing jobs | Large-scale scheduled/queued compute jobs |
| [AWS Outposts](./AWS%20Outposts.md) | AWS infrastructure on-premises | Run AWS services in your own data center |

## How to choose

```
Need full OS-level control → EC2
Want to deploy a web app without managing servers → Elastic Beanstalk
Simple website or small project, predictable pricing → Lightsail
Process large queues of compute jobs efficiently → AWS Batch
Need AWS services running in your own data center → Outposts
```

**Note:** Serverless compute (Lambda, Fargate) is in the [Serverless](../Serverless/) category.
