Task 3.1

Deploying and Operating in AWS

How you interact with AWS, how you define infrastructure as code, and what deployment model fits different workloads.

🖶
Web UI

AWS Management Console

Point-and-click GUI at aws.amazon.com. Good for exploring services, one-time tasks, viewing dashboards. Not ideal for repetitive work — hard to reproduce and automate.

>
Terminal commands

AWS CLI

Command-line interface for controlling AWS from a terminal. Ideal for scripting, automation, and piping output into other tools. How operations engineers work day-to-day.

💻
In-code API access

AWS SDKs

Libraries for Python, JavaScript, Java, .NET, Go, and more. Let your application call AWS APIs directly. Used when an app needs to upload files to S3, query DynamoDB, or invoke Lambda.

📄
Infrastructure as Code

AWS CloudFormation

Define infrastructure in JSON/YAML templates and deploy reproducible stacks automatically. Version-controlled, repeatable, consistent. A stack is a collection of AWS resources managed as a unit.

🚀
Managed app platform

Elastic Beanstalk

Upload code; AWS provisions EC2 instances, load balancers, and auto-scaling. Good for traditional web apps (Python, Node.js, Java, PHP, Docker) without managing servers directly.

Deployment models

All in AWS

Cloud

Everything runs in AWS. No on-premises infrastructure. Maximum cloud benefits. Used by startups and cloud-native applications.

Split between AWS and on-premises

Hybrid

Some resources in AWS, some on-premises. Most common for established companies migrating to cloud. Requires VPN or Direct Connect connectivity.

Stay in your data center

On-premises

Everything in the company's own data centers. AWS Outposts brings AWS hardware on-premises for compliance or latency requirements while keeping the AWS tooling experience.

Task 3.2

AWS Global Infrastructure

AWS organizes its physical infrastructure into Regions, Availability Zones, and Edge Locations — each serving a different purpose in building highly available, low-latency architectures.

Region

Geographic area with 2+ Availability Zones

Fully independent — data doesn't leave a Region without your explicit action. Choose based on compliance, user proximity, service availability, and price. Over 30 Regions worldwide.

AZ

One or more isolated data centers within a Region

Separate power, cooling, and physical location (miles apart). High-bandwidth, low-latency fiber between AZs in the same Region. Multi-AZ deployments survive single AZ failure. This is how you achieve high availability.

Edge

400+ global points of presence for CloudFront and Route 53

Far more than Regions or AZs. Positioned close to end users in major cities worldwide. Content is cached here — users get data from the nearest edge instead of traveling to the origin Region.

Local Zone

Compute closer to specific metropolitan areas

Extensions of Regions placed in large cities. Single-digit millisecond latency for applications serving users in a specific metro. Used for media rendering, gaming, real-time apps.

Wavelength

AWS compute inside 5G telecom networks

Ultra-low latency for 5G mobile devices. AWS services embedded in carrier networks. Used for autonomous vehicles, AR/VR, real-time mobile applications.

Outposts

AWS hardware racks in your own data center

Run native AWS services on-premises using the same APIs and tools as in the cloud. For workloads with compliance or latency requirements that must stay on-premises.

ⓘ

Region selection: the priority order

1. Compliance/data residency — overrides everything else. If regulations require data in a specific country, it goes there. 2. Latency — deploy near your users. 3. Service availability — not all services exist in all Regions. 4. Pricing — same service costs vary by Region; usually a secondary factor.

Task 3.3

Compute Services

From full virtual servers to serverless functions — AWS offers multiple ways to run code depending on how much control and responsibility you want.

🖥
Virtual servers

Amazon EC2

Full virtual server — you control the OS, software, and configuration. Best for long-running apps, legacy lift-and-shift, or anything needing specific OS configuration. Instance types: General Purpose, Compute, Memory, Storage, GPU Optimized.

λ
Serverless functions

AWS Lambda

Write a function; AWS runs it when triggered (S3 upload, HTTP request, schedule, queue message). No servers to manage. Scales automatically. Pay per millisecond of execution. Max 15-minute timeout.

▣
Managed containers

Amazon ECS / EKS

ECS: AWS's container orchestration for Docker. EKS: Managed Kubernetes — if you already use K8s, EKS lets you run it on AWS without managing the control plane. Both work with EC2 or Fargate.

🏭
Serverless containers

AWS Fargate

Run containers without managing EC2 instances. Define container specs; Fargate handles the infrastructure. Works with both ECS and EKS. The serverless option for containers.

🚀
Managed app platform

Elastic Beanstalk

Upload code; AWS manages EC2, load balancing, and auto-scaling. Best for traditional web applications when you want platform management without choosing each service individually.

▣
Parallel batch jobs

AWS Batch

Manages and runs large numbers of batch computing jobs. You define the jobs; Batch handles provisioning, queuing, and execution. For scientific simulations, genomics, financial risk models.

EC2 pricing models

On-Demand
No commitment

Pay by the hour or second. Start and stop anytime. Most expensive per hour.

Use when: Unpredictable workloads, short-term projects, development environments
Reserved Instances
Up to 72% off

Commit to 1 or 3 years for a specific instance type in a Region. Standard or Convertible options.

Use when: Steady-state workloads running continuously — production databases, stable web apps
Savings Plans
Up to 66% off

Commit to a $/hour spending amount. Applies to EC2, Lambda, and Fargate. More flexible than RIs.

Use when: You want RI-level discounts but need flexibility to change instance types
Spot Instances
Up to 90% off

Unused AWS capacity. Can be interrupted with 2 minutes notice. Lowest cost.

Use when: Batch processing, video rendering, CI/CD — workloads that can handle interruption
Dedicated Hosts
Physical server

Physical server dedicated to your use. Most expensive. Required for certain BYOL licenses.

Use when: Compliance requires dedicated hardware, or BYOL software licensed per physical core
Task 3.4

Database Services

AWS provides purpose-built database services for relational data, high-scale NoSQL, in-memory caching, analytics warehousing, and specialized use cases.

Managed relational

Amazon RDS

Runs MySQL, PostgreSQL, MariaDB, Oracle, SQL Server. AWS handles OS patching, automated backups, Multi-AZ replication, and failover. You manage schema, queries, and users.

High-performance relational

Amazon Aurora

AWS-built engine compatible with MySQL and PostgreSQL. Up to 5x faster than MySQL. Stores 6 copies across 3 AZs automatically. Aurora Serverless scales to zero when idle.

Serverless NoSQL

Amazon DynamoDB

Key-value and document store. Serverless — no instances to manage. Single-digit millisecond latency at any scale. Global Tables replicate across Regions automatically.

In-memory cache

Amazon ElastiCache

Managed Redis or Memcached. Cache frequently-read data in memory for microsecond latency. Dramatically reduces database load for high-read workloads.

Data warehouse (OLAP)

Amazon Redshift

Columnar storage optimized for analytical queries over massive datasets. For business intelligence, historical analysis, and OLAP workloads. Not for transactional applications.

Graph database

Amazon Neptune

Optimized for highly connected data — relationships are as important as the data itself. Social networks, fraud detection, recommendation engines, knowledge graphs.

⚠

OLTP vs OLAP — know the difference

OLTP (transactional) — many small, fast operations. Customer places order, record inserted. Use RDS or DynamoDB. OLAP (analytical) — complex queries aggregating large historical data. "Sales by region last year?" Use Redshift. Using the wrong database for the wrong workload is a common exam scenario.

Task 3.5

Network Services

How traffic reaches your application, how you route it intelligently, and how you connect your on-premises network to AWS.

DNS and traffic routing

Amazon Route 53

Managed DNS with health checks and routing policies. Latency routing sends users to the lowest-latency Region. Failover routing redirects to backup when primary is unhealthy. Weighted routing splits traffic by percentage.

HTTP/HTTPS load balancing

Application Load Balancer

Layer 7 load balancer. Routes requests by URL path, hostname, headers. One ALB can route /api to one service and /images to another. The standard choice for web applications and microservices.

High-performance TCP/UDP

Network Load Balancer

Layer 4 load balancer. Millions of requests/second with ultra-low latency. No HTTP routing rules. For gaming, financial trading, real-time systems requiring maximum throughput.

Global CDN

Amazon CloudFront

Caches content at 400+ Edge Locations worldwide. Users get data from nearby edge nodes instead of your origin Region. Integrates with WAF and Shield for edge security. Provides HTTPS via ACM certificates.

Managed API

AWS API Gateway

Create, secure, and monitor REST and WebSocket APIs. Routes requests to Lambda, EC2, or other backends. Handles authentication, throttling, caching. The front door for serverless applications.

Multi-VPC hub

AWS Transit Gateway

Central hub connecting multiple VPCs, VPNs, and Direct Connect. Hub-and-spoke model. Each network connects once to Transit Gateway instead of building a mesh of individual peering connections.

Dedicated private connection

AWS Direct Connect

Private fiber from your data center to AWS. No internet. Consistent, predictable bandwidth. For high-volume data transfer and latency-sensitive hybrid applications. Takes weeks to provision.

Global routing optimization

AWS Global Accelerator

Routes user traffic through the AWS global backbone network (not the public internet). Provides static IPs. Better for dynamic content, non-HTTP protocols, and consistent global performance.

Task 3.6

Storage Services

Object storage, block storage, file systems, archival, and physical data transfer — each optimized for different access patterns and cost profiles.

Object storage

Amazon S3

Store any file type in globally-named buckets. 11 nines of durability (replicated across 3 AZs). Unlimited capacity. Lifecycle policies move objects between tiers automatically based on age. Static website hosting included.

Block storage for EC2

Amazon EBS

Persistent disk volumes attached to EC2 instances. Data survives instance restarts. Snapshots to S3 for backups. One volume per instance (generally). SSD types for performance; HDD types for throughput/cost.

Shared file system

Amazon EFS

NFS file system mountable by multiple Linux EC2 instances simultaneously. Elastic — grows and shrinks automatically. For shared content, web server farms, ML training data accessed by many instances.

Windows file server

Amazon FSx

FSx for Windows: SMB protocol, AD-integrated. FSx for Lustre: High-performance parallel file system for HPC and ML, integrates with S3. Managed Windows/Lustre without the operational overhead.

Physical data transfer

AWS Snow Family

Physical devices for moving large datasets to/from AWS when internet transfer is impractical. Snowcone: 8–14 TB, ruggedized. Snowball Edge: ~80–210 TB. Snowmobile: 100 PB truck. Use for >10 TB migrations.

Hybrid cloud storage

AWS Storage Gateway

On-premises applications see local NFS/SMB/iSCSI storage; behind the scenes, data is stored in S3 or Glacier. Bridge for extending on-premises storage to the cloud and migrating backups.

S3 storage class tiers

Task 3.7

AI/ML and Analytics Services

Pre-built AI capabilities and data processing tools. For most AI services, you call an API — no ML expertise required. For custom models, use SageMaker.

AI and Machine Learning

🔬

Amazon SageMaker

Build custom ML models

End-to-end platform for training and deploying custom ML models on your own data. For when pre-built services don't fit.

📷

Amazon Rekognition

Image and video analysis

Object detection, facial analysis, text in images, content moderation, celebrity recognition, activity detection.

📄

Amazon Comprehend

Text analysis (NLP)

Sentiment analysis, entity extraction, key phrases, language detection, PII detection. Analyze customer feedback at scale.

🌐

Amazon Translate

Language translation

Real-time machine translation between 75+ languages. For multilingual apps and content localization.

💬

Amazon Lex

Conversational AI / Chatbots

Build chatbots and voice interfaces using the same technology as Alexa. Understands natural language intent and manages conversation flow.

🔊

Amazon Polly

Text to speech

Converts text to lifelike speech audio. For voice-enabled apps, accessibility features, audio content.

🎤

Amazon Transcribe

Speech to text

Converts audio/video to text transcript. Speaker identification, custom vocabulary, live transcription. For call analysis, captions, meeting notes.

🗃

Amazon Textract

Extract data from documents

OCR plus structure extraction — reads form fields, tables, and handwriting from scanned PDFs and images. Automates invoice and form processing.

📊

Amazon Forecast

Time-series forecasting

Predicts future values from historical data. Demand forecasting, inventory planning, financial projections.

🔎

Amazon Kendra

Enterprise document search

ML-powered search that returns direct answers from documents, not just a list of links. For internal knowledge bases and policy search.

🛍

Amazon Personalize

Recommendations

Real-time personalized recommendations from user behavior data. "Customers who bought this…" product suggestions, personalized content feeds.

Analytics Services

Query S3 with SQL

Amazon Athena

Serverless SQL queries directly against data in S3. No ETL, no database loading, no cluster. Pay per query. Best for ad-hoc analysis of log files and datasets already stored in S3.

ETL pipelines

AWS Glue

Serverless ETL service. Crawlers catalog your data sources automatically. Jobs transform and move data between S3, Redshift, and RDS. Includes a central Data Catalog used by Athena and Redshift.

Real-time data streaming

Amazon Kinesis

Ingest and process data streams in real time — milliseconds of latency. Data Streams: ingest and store. Firehose: load to S3/Redshift without code. Analytics: SQL on live streams.

BI dashboards

Amazon QuickSight

Business intelligence visualization tool. Connects to Redshift, RDS, S3, Athena, and third-party sources. Creates interactive dashboards and charts for business stakeholders.

Big data / Hadoop / Spark

Amazon EMR

Managed big data platform for Hadoop, Spark, Hive, and Presto. For large-scale ETL, ML on big data, or migrating existing Spark/Hadoop workloads to AWS.

Data lake setup

AWS Lake Formation

Automates building and securing a data lake on S3. Handles data ingestion, cataloging, security policies, and access controls. Reduces weeks of setup to days.

Task 3.8

Other In-Scope Services

Application integration, developer/CI-CD tools, migration, end-user computing, and management services that round out the Domain 3 service coverage.

Application integration (messaging & events)

Pub/Sub notifications

Amazon SNS

Publish a message to a topic; all subscribers receive it simultaneously. Fan-out — one event triggers email, SMS, Lambda, SQS, and HTTP endpoints at the same time. One-to-many.

Message queue

Amazon SQS

Managed message queue for async decoupling. Messages wait until a consumer is ready. Absorbs traffic spikes and prevents message loss if downstream services are slow or temporarily down. One-to-one.

Event bus

Amazon EventBridge

Routes events from AWS services, SaaS apps, and your own applications to targets based on rules. Unlike SNS, it can filter by event content — route only specific event types to specific targets.

Visual workflow orchestration

AWS Step Functions

Coordinate multi-step processes as state machines. Each step calls a Lambda function or AWS service. Built-in error handling and retry logic. For order fulfillment, document processing, and approval workflows.

Developer tools — CI/CD pipeline

ServiceRole in the pipeline
AWS CodeCommitManaged private Git repository for source code storage
AWS CodeBuildManaged build service — compiles code, runs tests, produces artifacts
AWS CodeDeployAutomates deployments to EC2, Lambda, and ECS; handles rollbacks
AWS CodePipelineOrchestrates the full workflow: commit → build → test → deploy

Migration services

Database migration

AWS DMS

Migrates databases to AWS with minimal downtime. Source database stays live during replication. Supports same-engine and cross-engine migrations (Oracle to PostgreSQL, etc.).

Server migration (lift & shift)

AWS Application Migration Service

Continuously replicates on-premises servers to AWS in the background. Cutover launches EC2 instances with minimal downtime. The standard path for lift-and-shift server migrations.

End-user computing & management

Virtual desktops

Amazon WorkSpaces

Full Windows or Linux cloud desktops accessible from any device. Data stays in AWS. For remote workers, call center agents, and compliance scenarios requiring data not to leave AWS.

Application streaming

Amazon AppStream 2.0

Stream individual desktop applications (not full desktops) to any browser. For partners or contractors who need access to one specific app without a full virtual desktop.

Fleet management

AWS Systems Manager

Manage EC2 fleets: patch automation, Session Manager (SSH without open ports), Run Command (scripts across many instances), Parameter Store (config values). Operational tool for infrastructure management.

Multi-account governance

AWS Control Tower

Automates setup of a well-architected multi-account "landing zone." Pre-configures Accounts, guardrails, and logging following AWS best practices. Start your multi-account environment correctly from day one.

Cloud contact center

Amazon Connect

Cloud-based call center service. Define call flows, manage agents, record calls. Integrates with Lex for IVR chatbots and Comprehend/Transcribe for call sentiment analysis.

Exam Prep

Practice Questions

Service selection scenarios — the most common format across Domain 3 questions.

Q1. A startup runs its production web application on EC2 On-Demand instances. After 12 months, their traffic patterns are stable and consistent. What change reduces compute costs most significantly?
A Switch to Spot Instances
B Purchase 1-year Reserved Instances for the production workload
C Move the application to AWS Lambda
D Enable Auto Scaling to reduce the number of instances
Answer: B. Stable, predictable traffic is the ideal use case for Reserved Instances — up to 72% discount over On-Demand for a 1-year commitment. Spot Instances can be interrupted with 2-minute notice, making them unsuitable for customer-facing production workloads. Lambda works for event-driven functions, not persistent web servers. Auto Scaling adjusts capacity to demand but doesn't change the pricing model.
Q2. A company serves users across North America, Europe, and Asia-Pacific. Static content (images, CSS, JS) is stored in S3 in us-east-1. Users in Europe and Asia report slow page loads. What is the most effective solution?
A Move the S3 bucket to eu-central-1 to be more central
B Enable S3 Cross-Region Replication to all regions
C Distribute content using Amazon CloudFront
D Use a larger EC2 instance to serve the content faster
Answer: C. CloudFront caches static content at 400+ Edge Locations worldwide. Users in Europe and Asia retrieve content from nearby edge nodes instead of making a cross-ocean round trip to us-east-1. Option A helps Europe but hurts Asia. Option B copies data to Regions, not Edge Locations, and requires routing logic. EC2 instance size doesn't address network distance latency.
Q3. An application receives image uploads from users and needs to automatically check each upload for inappropriate content before displaying it. The check should happen without running a server 24/7. What architecture fits?
A EC2 instance that polls S3 for new uploads every minute
B S3 triggers a Lambda function on each upload; Lambda calls Amazon Rekognition to analyze the image
C Elastic Beanstalk application that processes uploads
D Amazon ECS cluster running a content moderation service
Answer: B. S3 events trigger Lambda functions automatically — no polling, no always-on server. Lambda calls Rekognition's content moderation API and takes action based on the result. This is a classic event-driven serverless pattern. EC2 and ECS require running servers. Beanstalk manages server infrastructure and adds complexity for this simple event-driven task.
Q4. A company's DynamoDB database serves a product catalog that gets 2 million reads per day. 80% of reads are for the same 200 top products. Database costs are high. What reduces cost while maintaining low latency?
A Move the product catalog to Amazon Redshift
B Use Amazon S3 to store product data instead
C Add Amazon ElastiCache in front of DynamoDB to cache the top products
D Enable DynamoDB auto-scaling
Answer: C. Caching the top 200 products in ElastiCache means 80% of reads never hit DynamoDB — dramatically reducing read capacity units consumed and therefore cost. Microsecond cache hits are faster than DynamoDB's single-digit millisecond responses. Redshift is for analytical workloads, not product catalog lookups. S3 is not a database. Auto-scaling helps with capacity management but doesn't reduce per-request costs.
Q5. A data analyst needs to run occasional SQL queries against 2 years of application log data stored in S3 as compressed JSON files. They want to query without provisioning any database or ETL pipeline. Which service is most appropriate?
A Amazon Redshift
B Amazon RDS
C Amazon Athena
D AWS Glue
Answer: C. Athena is serverless SQL for data already in S3. No cluster, no ETL, no data loading — point it at the S3 prefix and query immediately. Pay per query. Redshift requires loading data into a cluster. RDS is a transactional database — requires schema definition and data loading. Glue builds ETL pipelines; it's a tool for moving and transforming data, not for querying.
Q6. A company collects 500,000 IoT sensor readings per minute and needs to detect anomalies within 3 seconds of data arrival. Which service processes this in real time?
A Amazon Kinesis Data Streams
B Amazon Athena
C AWS Glue
D Amazon QuickSight
Answer: A. Kinesis ingests and processes streaming data in real time — milliseconds to seconds of latency. Kinesis Data Analytics can run SQL against the live stream for anomaly detection. Athena and Glue process data in batches — unsuitable for sub-3-second requirements. QuickSight is a visualization tool, not a stream processing service.
Q7. A company has 800 TB of archive data on physical tape that they want to move to AWS. They also need to continue writing new archive data to what appears to be a tape library (their backup software can't be changed). What combination of services addresses this?
A Direct Connect to upload the archive; EFS for ongoing writes
B Snowball Edge to migrate the existing 800 TB; Storage Gateway Tape Gateway for ongoing backup software integration
C S3 Transfer Acceleration for both migration and ongoing writes
D S3 for migration; EBS for ongoing writes
Answer: B. Snowball Edge handles the physical 800 TB migration — faster and more practical than internet transfer. Storage Gateway's Tape Gateway presents virtual tape drives to the backup software over the network; the software writes to "tapes" normally, but data goes to S3 and Glacier. This is precisely what Tape Gateway is designed for — transparent cloud backup without changing backup software.
Q8. A company wants to automatically analyze customer support call recordings to detect customer sentiment and identify which product issues are most commonly mentioned. Which two services would work together for this?
A Amazon Polly and Amazon Kendra
B Amazon Transcribe and Amazon Comprehend
C Amazon Lex and Amazon Rekognition
D Amazon Textract and Amazon Translate
Answer: B. Transcribe converts audio recordings to text transcripts. Comprehend analyzes the text for sentiment (positive/negative/neutral) and entity extraction (which products are mentioned). Polly generates speech from text (opposite direction). Kendra is document search. Lex builds chatbots; Rekognition analyzes images. Textract reads printed documents; Translate changes language — none of these fit audio + sentiment analysis.
