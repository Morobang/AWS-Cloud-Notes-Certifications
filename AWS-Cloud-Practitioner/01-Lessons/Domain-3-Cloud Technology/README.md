# Domain 3: Cloud Technology and Services

## What this domain covers

Domain 3 is the largest on the Cloud Practitioner exam at **34% (~23 questions)**. It covers the actual AWS services — what they do, when you use each one, and how they fit together to build real workloads.

The focus is breadth over depth. You don't need to know how to configure services in detail, but you do need to know which service to reach for in a given scenario.

## Key terms

**Serverless** — You provide the code; AWS manages all the infrastructure. No servers to configure, patch, or scale. You pay per invocation.

**Managed service** — AWS handles the operational work (patching, backups, failover). You handle your data and configuration.

**Relational database** — Data stored in tables with rows and columns. Uses SQL. Examples: MySQL, PostgreSQL.

**NoSQL database** — Data stored in a flexible format (key-value, document, graph). Better for large scale or variable data shapes.

**Object storage** — Data stored as objects (files + metadata). Accessed via HTTP. No file system hierarchy.

**Block storage** — Data stored as fixed-size blocks, like a hard drive. Attached to and managed by a server.

**CDN (Content Delivery Network)** — Copies of your content placed in locations around the world so users can retrieve data from nearby servers instead of traveling to your origin.

**IaC (Infrastructure as Code)** — Defining cloud infrastructure in code files that can be version-controlled, repeated, and automated.
