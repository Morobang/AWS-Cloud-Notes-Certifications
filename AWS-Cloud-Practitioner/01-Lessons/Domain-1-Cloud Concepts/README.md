# Domain 1: Cloud Concepts

## What is "the cloud"?

Before cloud computing, running a website or application meant buying servers — powerful computers that sit in a room, run 24/7, and require maintenance, security, electricity, and someone to manage them. If your business grew, you bought more servers. If it slowed down, you still paid for all that hardware.

Cloud computing changes the model. Instead of owning computing infrastructure, you rent it from a provider like Amazon Web Services (AWS) — paying for what you actually use, scaling up when you need more, and scaling down when you don't.

The everyday analogy: you don't build a power plant to get electricity. You plug in, use what you need, and pay at the end of the month. AWS is the power plant for computing.

## Key terms

**Data center** — A building full of servers. AWS has thousands of data centers globally.

**Region** — A geographic area (like US East, Europe Ireland, Asia Pacific Tokyo) where AWS has data centers. There are 30+ regions.

**Availability Zone (AZ)** — A cluster of data centers within a region, physically separated from other AZs for fault tolerance. Every region has at least 3 AZs.

**On-premises** — Keeping IT in your own building. The traditional approach.

**CapEx** — Capital Expenditure. Buying hardware or software upfront. A one-time large cost.

**OpEx** — Operating Expenditure. Paying for services over time, like a monthly subscription. Cloud billing is OpEx.
